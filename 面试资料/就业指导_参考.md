```python
# 就是入职之后，基本上每天的工作内容是什么？工作的压力大吗？
刚毕业的运维每天的工作可能是：
完成领导分配的工作，每周可能有一到两个目标，需要完成，必输部署服务、帮助业务上的服务实现容器迁移，编写一些脚本、处理日常工单和报警；

熟悉环境，接需求，解决问题，压力大不大随机，前期不熟肯定大，自己要拼命努力度过试用期；

# 为什么面试官会反感培训出来的人，如果在面试的时候，面试的人要是问了，咋回答？
因为培训出来的人要的太多、太水，比如想要一个10年工作经验的人，结果来了一个培训出来的，面试官的落差比较大；不怕谈崩，大不了换下一家接着面；

# 新进入一家公司做运维岗，通过怎么样流程和方向，尽快的拿捏好工作，让自己尽快胜任工作，安全度过试用期呢？
# 初入公司涉及到人际着重关注点是什么，尽量是自己部门，协同部门研发，测试，产品熟悉吗？
把现有的架构熟悉了，满足日常工作，正常就可以度过试用期，一般新入职一家公司，会发现已经存在的一些问题，想要解决策略，找领导商量，可以提前度过试用期；
一般情况下，把自己的本质工作做好，和自己部门的领导同事打好关系就行，慢慢熟悉；
上面说了，认真听需求，认真干活，解决问题，产出技术文档，留作备用，以及主动多探索技术架构，主动找领导，分担任务，增大自己的价值；
刚开始也就和自己部门的熟悉就好了，其他部门的会逐步的合作，逐步的认识。
```
