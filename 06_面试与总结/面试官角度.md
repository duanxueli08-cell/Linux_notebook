HR 阶段的目标就是：**筛掉明显不符、夸大简历、沟通障碍或动机不匹配的人**，把真正有潜力的候选人交给技术团队深入评估。

作为招聘 **Linux 系统运维工程师** 的人事专员（HR），虽然你可能不具备深入的技术背景，但你的目标是**初步筛选出具备基本技术素养、沟通能力、稳定性与岗位匹配度的候选人**。因此，问题设计应聚焦在以下几个维度：

---

### 一、基础信息与职业动机（通用类）

1. **请简单介绍一下你自己，以及你为什么想从事 Linux 系统运维工作？**  
   → 考察表达能力、职业动机是否清晰。

2. **你之前的工作/项目中，主要使用哪些 Linux 发行版？（如 CentOS、Ubuntu、Debian、Rocky Linux 等）**  
   → 判断实际接触经验，避免只学过理论。

3. **你应聘这个岗位，最看重哪些方面？（技术成长？稳定性？团队氛围？）**  
   → 了解求职动机与公司文化是否匹配。

---

### 二、技术能力初筛（非深度，但需真实）

> 💡 提示：这些问题不需要 HR 懂答案，但可通过候选人回答的**流畅度、关键词使用、举例具体性**判断真伪。

4. **你在日常工作中最常使用的 Linux 命令有哪些？能举几个例子说明用途吗？**  
   → 看是否熟悉 `grep`、`awk`、`sed`、`top`、`df`、`netstat/ss`、`systemctl` 等基础命令。

5. **如果服务器突然变慢或无法访问，你会怎么排查？**  
   → 观察是否有系统性思路（如：查负载 → CPU/内存/磁盘 → 网络 → 日志）。

6. **你有没有配置或维护过 Web 服务（如 Nginx/Apache）？数据库（如 MySQL/PostgreSQL）？**  
   → 判断是否具备常见服务部署经验。

7. **你用过 Shell 脚本吗？写过哪些自动化任务？**  
   → 考察是否具备基础自动化意识（如日志清理、备份脚本等）。

8. **你了解什么是 RAID、LVM、防火墙（iptables/firewalld）吗？**  
   → 判断是否接触过系统底层或安全配置。

---

### 三、工具与流程意识（加分项）

9. **你是否使用过版本控制工具（如 Git）？是否接触过 CI/CD 或 DevOps 工具链？**  
   → 现代运维常需协作开发，Git 是基本要求。

10. **你有没有用过监控工具（如 Zabbix、Prometheus、Nagios）或日志系统（如 ELK）？**  
    → 判断是否具备运维体系化思维。

11. **你是否接触过云平台（如阿里云、AWS、腾讯云）或虚拟化技术（如 Docker、KVM）？**  
    → 很多企业已上云，相关经验是重要加分项。

---

### 四、软技能与工作风格

12. **运维工作经常需要处理紧急故障，甚至半夜响应。你如何看待这种工作节奏？**  
    → 考察抗压能力与职业认知。

13. **你过去是否参与过团队协作的运维项目？如何与其他同事（如开发、测试）配合？**  
    → 判断沟通协作能力。

14. **你最近半年有学习新的技术或考取相关证书吗？（如 RHCE、CKA、阿里云 ACA/ACP）**  
    → 评估学习主动性。

---

### 五、反向验证（防“包装”）

- 如果对方说“精通 Linux”，可追问：  
  > “能不能简单说一下从你按下电源到系统登录完成，Linux 启动过程经历了哪些关键步骤？”  
  → 真实有经验的人通常能说出 BIOS → bootloader（GRUB）→ kernel → init/systemd → login。

- 如果简历写“负责高可用集群”，可问：  
  > “你用的是什么高可用方案？Keepalived 还是 Pacemaker？主备切换时怎么保证数据一致？”  
  → 看是否只是“听过”还是真正操作过。

---

### ✅ HR 面试小贴士：

- **不要求你判断技术对错**，但要关注：
  - 回答是否**具体**（有场景、有命令、有结果）
  - 是否**回避问题**或**过度模糊**
  - 是否**夸大其词**（如“我一个人维护上千台服务器”但无细节支撑）

- 可准备一份**技术关键词清单**（如：Shell、Nginx、RAID、Docker、Zabbix、systemd），看候选人是否自然提及。

- 对于初级岗位，重点看**学习能力+基础扎实**；对于中级以上，看**故障处理经验+架构意识**。

### 工作地点：[例如：上海/北京/远程]

---

### 【岗位职责】
- 负责 Linux 服务器（CentOS/Ubuntu/Rocky 等）的部署、配置、监控与日常维护；
- 编写 Shell/Python 脚本，实现系统部署、日志分析、备份、监控等自动化任务；
- 配置和管理 Web 服务（Nginx/Apache）、数据库（MySQL/PostgreSQL/Redis）及中间件；
- 处理系统故障、性能瓶颈及安全事件，保障服务高可用性（99.9%+ SLA）；
- 参与 CI/CD 流程建设，配合开发团队完成应用发布与环境管理；
- 管理云平台资源（如阿里云、AWS、腾讯云）或私有化虚拟化环境（KVM/Docker/K8s）；
- 制定并执行系统安全策略（防火墙、SSH 安全、漏洞修复、权限控制）；
- 编写运维文档，包括操作手册、应急预案、架构图等。

---

### 【任职要求】
#### 必备条件：
- 计算机相关专业大专及以上学历，2 年以上 Linux 运维经验；
- 熟悉主流 Linux 发行版，能熟练使用常用命令及系统工具（grep/sed/awk/top/iostat 等）；
- 掌握 Shell 脚本编写，了解 Python 或 Go 者优先；
- 熟悉 TCP/IP 网络原理，能排查网络连通性、端口、DNS 等问题；
- 有 Nginx、MySQL、Redis、RabbitMQ 等常见服务的部署与调优经验；
- 了解基础安全防护措施（如 iptables/firewalld、fail2ban、用户权限管理）；
- 具备良好的沟通能力、责任心强，能接受轮班或紧急故障响应。

#### 加分项（非必需但优先）：
- 有公有云（阿里云/AWS/腾讯云）实际运维经验；
- 熟悉 Docker、Kubernetes 容器化技术；
- 使用过 Zabbix/Prometheus/Grafana 等监控系统；
- 有 CI/CD 工具（Jenkins/GitLab CI）使用经验；
- 持有 RHCE、CKA、阿里云 ACP 等认证；
- 有高并发、高可用架构（如负载均衡、主从复制、集群）实战经验。

## 二、Linux 运维岗位技术面试题库（分层级）

> 供技术面试官使用，HR 可作为参考了解考察重点

---

### 🔹 初级（0–2 年经验）
1. 如何查看系统 CPU、内存、磁盘使用情况？分别用哪些命令？
2. 写一个 Shell 脚本，每天凌晨 2 点自动备份 `/data` 目录到 `/backup`。
3. 如何查看某个端口是否被占用？如何杀死占用该端口的进程？
4. 解释 `chmod 755 filename` 的含义。
5. 如何设置开机自启动一个服务？（systemd vs init.d）
6. 如何查找系统中所有大于 100MB 的文件？
7. SSH 登录失败可能有哪些原因？如何排查？

---

### 🔹 中级（2–5 年经验）
1. 描述一次你处理过的线上服务器宕机或高负载故障，你是如何定位和解决的？
2. 如何实现 Nginx 的反向代理和负载均衡？配置关键参数有哪些？
3. MySQL 主从复制原理是什么？如果主从延迟严重，你会怎么排查？
4. 请解释 LVM 的作用，并演示如何扩容一个逻辑卷。
5. 如何用 `iptables` 或 `firewalld` 限制某 IP 访问 22 端口？
6. 你如何做系统安全加固？（至少列出 5 项措施）
7. 如何监控一台服务器的健康状态？你会采集哪些指标？

---

### 🔹 高级（5 年+ / SRE 方向）
1. 设计一个高可用 Web 架构（含负载均衡、应用层、数据库、缓存），画出拓扑图并说明关键点。
2. 如何实现零停机发布（Blue-Green / Canary）？结合你的项目经验说明。
3. 容器化后，传统运维角色发生了哪些变化？你如何适应？
4. 如何用 Prometheus + Alertmanager 实现自定义告警？举例说明。
5. 当系统出现“CPU 使用率不高但响应慢”时，可能是什么原因？（提示：I/O wait、锁竞争、网络延迟）
6. 如何审计 Linux 系统的操作日志？如何防止日志被删除？
7. 谈谈你对 Infrastructure as Code（IaC）的理解，是否使用过 Ansible/Terraform？

---

### 🔹 场景题（考察实战思维）
- “凌晨 3 点收到报警：API 响应时间从 200ms 升到 5s，你第一步做什么？”
- “开发说‘在我本地能跑’，但测试环境部署失败，你怎么协助排查？”
- “公司要将 50 台物理机迁移到阿里云，你会制定怎样的迁移方案？”
